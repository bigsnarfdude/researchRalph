# minif2f-lean4-opus4-proofs

223 verified Lean 4 proofs on the MiniF2F benchmark, generated by Claude Opus 4.6 via autonomous multi-agent search (researchRalph v4.4). Every proof passes `lake env lean` against Mathlib. 13 proofs include full `<think>` chain-of-thought reasoning traces.

## What's in the box

- **223 verified proofs** covering 223/244 MiniF2F problems (91.4%)
- **13 gold records**: verified proof + Opus `<think>` reasoning trace (R1-style)
- **122 reasoning-style**, **101 simple-style** proofs. Zero shotgun. Zero sorry.
- All proofs compile in ≤32M heartbeats (no brute-force timeouts)
- No linter suppressions (`linter.all false` filtered out)

## Quality filters applied

This dataset was aggressively filtered before publication:

| Filter | Dropped | Reason |
|--------|---------|--------|
| `passed = false` | 13 | Failed `lake env lean` verification |
| `sorry` / `admit` | 0 | Already caught by sorry→score downgrade |
| `maxHeartbeats > 32M` | 8 | Brute-force proofs that take minutes to compile |
| `linter.all false` | 0 | Already caught by heartbeat filter |
| Polluted thinking traces | 5 | Traces mostly about other problems |

**Audit**: Every record was graded by Claude Opus via `claude -p`. 94.3% clean on style/tier/quality/trace-relevance. Remaining 5.7% are borderline style classifications (simple vs reasoning on 4-line proofs), not data errors.

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `problem_id` | string | MiniF2F problem identifier |
| `lean_code` | string | Complete Lean 4 file (imports + theorem + proof) |
| `score` | int | Always 5 (verified) in this filtered release |
| `passed` | bool | Always true in this filtered release |
| `error_type` | string | Always "verified" in this filtered release |
| `tier` | string | `gold` (verified + Opus trace) or `verified` (no trace) |
| `experiment` | string | Which RRMA experiment generated this proof |
| `thinking_trace` | string or null | `<think>...</think>` CoT from Claude Opus 4.6 |
| `tactic_style` | string | `reasoning` (structured) or `simple` (≤3 tactic lines) |
| `thinking_chars` | int | Length of thinking trace (0 if none) |
| `proof_chars` | int | Length of proof body after `:= by` |
| `model` | string | Always `claude-opus-4-6` |
| `source` | string | Always `rrma-lean-v4` |

### Tactic Styles

| Style | Count | Description |
|-------|-------|-------------|
| `reasoning` | 122 | Multi-step proof with `have`, `rcases`, `calc`, `induction`, `obtain`, `suffices` |
| `simple` | 101 | ≤3 tactic lines — `ring`, `omega`, `norm_num`, `simp`, `native_decide` |

## Source

- **Agent framework**: [researchRalph](https://github.com/bigsnarfdude/researchRalph) v4.4
- **Model**: Claude Opus 4.6 (via Anthropic API)
- **Verification**: `lake env lean` against miniF2F-lean4 + Mathlib
- **Hardware**: RTX 4070 Ti (nigel server, BIRS)
- **Generation**: 82 experiments, 8 agents, March 2026
- **Audit**: Per-record grading by Claude Opus via `claude -p`

## Pipeline

```
RRMA v4.4 agents (Claude Opus 4.6)
    │
    ├── 82 experiments, 8 agents generate .lean files
    │
    ├── traces_to_sft.py → extract <think> chains from agent logs
    │        │
    │        └── relevance filter: reject traces about other problems
    │
    ├── score_attempts.py → lake env lean + proximity scoring (0-5)
    │
    ├── build_dataset.py → join scores + traces + lean code
    │        │
    │        ├── sorry filter: downgrade any proof with sorry
    │        └── trace pollution filter: reject contaminated traces
    │
    ├── dedup_dataset.py → 1 best proof per problem
    │        │
    │        └── ranking: no-sorry > score > trace > style > shortest
    │
    ├── audit_dataset.py → per-record Opus audit via claude -p
    │
    └── HF filter: drop failed + sorry + heartbeats>32M + linter hacks
                         │
                         └── minif2f_lean4_opus4_proofs.jsonl (this file)
```

## Intended Use

- **SFT training**: Fine-tune smaller models on verified proofs + thinking traces (R1-style)
- **Signal research**: Do CoT traces from frontier models help small models prove theorems?
- **Benchmark**: 223/244 MiniF2F coverage from pure multi-agent search (no training)
- **Proof mining**: Clean, compilable Lean 4 proofs for Mathlib-compatible projects

## Limitations

- Proofs verified by `lake env lean` but not human-reviewed for elegance
- 13 thinking traces is small for SFT — use as seed data, not sole training set
- MiniF2F is competition math — not representative of all theorem proving
- Some proofs use `native_decide` which is correct but opaque
- 21 problems excluded by quality filters (8 unsolved + 13 failed quality bar)

## Signal test

Fine-tuning Qwen2.5-7B-Instruct on RRMA traces showed perplexity improvement (13.59 → 10.36) but 0/30 pass@4 on held-out problems. The model learns the reasoning distribution but cannot yet generate verified proofs. More data or a larger base model is needed to close the generation gap.
